# Rain Forecast

> **ML-система прогнозирования осадков на следующий день на основе метеорологических наблюдений и методов машинного обучения.**

## 📖 О проекте

**Rain Forecast** — проект, посвященный решению задачи бинарной классификации, целью которой является прогнозирование вероятности выпадения осадков на следующий день на основе исторических метеорологических данных.

Работа охватывает полный цикл разработки модели машинного обучения: от исследования и подготовки данных до обучения классификатора, оценки качества модели и анализа наиболее значимых признаков. Особое внимание уделено обработке пропущенных значений, работе с категориальными признаками и построению информативного пространства признаков.

Проект был реализован с использованием открытого набора данных Kaggle и демонстрирует практический подход к решению задач прогнозирования погодных условий.

# 📊 Структура датасета

Каждая запись представляет собой ежедневное наблюдение погодных условий и включает следующие характеристики.

| Признак           | Описание                                                             |
| ----------------- | -------------------------------------------------------------------- |
| **Date**          | Дата наблюдения                                                      |
| **MinTemp**       | Минимальная температура воздуха                                      |
| **MaxTemp**       | Максимальная температура воздуха                                     |
| **Rainfall**      | Количество выпавших осадков (мм)                                     |
| **Evaporation**   | Испарение воды (мм)                                                  |
| **Sunshine**      | Продолжительность солнечного сияния (часы)                           |
| **WindGustDir**   | Направление наиболее сильного порыва ветра                           |
| **WindGustSpeed** | Скорость наиболее сильного порыва ветра                              |
| **WindDir9am**    | Направление ветра в 9:00                                             |
| **WindDir3pm**    | Направление ветра в 15:00                                            |
| **WindSpeed9am**  | Скорость ветра в 9:00                                                |
| **WindSpeed3pm**  | Скорость ветра в 15:00                                               |
| **Humidity9am**   | Влажность воздуха в 9:00                                             |
| **Humidity3pm**   | Влажность воздуха в 15:00                                            |
| **Pressure9am**   | Атмосферное давление в 9:00                                          |
| **Pressure3pm**   | Атмосферное давление в 15:00                                         |
| **Cloud9am**      | Облачность в 9:00                                                    |
| **Cloud3pm**      | Облачность в 15:00                                                   |
| **Temp9am**       | Температура воздуха в 9:00                                           |
| **Temp3pm**       | Температура воздуха в 15:00                                          |
| **RainToday**     | Наличие осадков в текущий день (Yes/No)                              |
| **RainTomorrow**  | **Целевая переменная** — прогноз выпадения осадков на следующий день |

# 🎯 Постановка задачи

Проект решает задачу **бинарной классификации**.

По совокупности метеорологических характеристик модель должна определить, будет ли дождь на следующий день.

**Входные данные**

* температура;
* влажность;
* давление;
* скорость и направление ветра;
* облачность;
* продолжительность солнечного сияния;
* количество осадков;
* другие погодные параметры.

**Выход модели**

```text
RainTomorrow

Yes  -> дождь ожидается
No   -> дождя не ожидается
```

# ⚙️ Pipeline проекта

Процесс построения модели состоит из нескольких этапов.

### 1. Исследовательский анализ данных

На первом этапе проводится анализ структуры датасета:

* изучение распределений признаков;
* поиск выбросов;
* анализ корреляций;
* исследование дисбаланса классов;
* визуализация погодных параметров.

---

### 2. Предобработка данных

Для повышения качества модели выполняются:

* обработка пропущенных значений;
* кодирование категориальных признаков;
* преобразование типов данных;
* масштабирование числовых признаков (при необходимости).

---

### 3. Обучение модели

После подготовки данных выполняется обучение модели классификации.

Особое внимание уделяется:

* качеству прогнозирования;
* устойчивости модели;
* способности к обобщению на новых данных.

---

### 4. Оценка качества

Полученная модель оценивается с использованием стандартных метрик классификации:

* Accuracy;
* Precision;
* Recall;
* F1-score;
* ROC-AUC (при наличии).

Это позволяет объективно оценить способность модели прогнозировать выпадение осадков.
